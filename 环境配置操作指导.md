# 环境配置操作指导

本环境配置指导手册仅限对于STAR的本地后端开发使用，此环境配置对于操作系统没有限制，通过使用docker容器镜像技术解决操作系统迁移问题，下面的步骤请在**windows操作系统**下使用。

## 开发软件——编辑器

推荐使用vscode作为开发软件

### 编辑器插件

1. python
2. markdownlint 
3. Markdown Prevoew Mermaid (若使用最新版vscode可直接在设置中启用mermaid，无须额外安装插件)
4. Container Tools 

## Docker Desktop 以及相关 docker 配置

**步骤1：** 浏览器或Microsoft Store直接下载Docker Desktop

**步骤2：** 启动Dcoker并注册登录，保持启运行状态

**步骤3：** 启动vscode，在开发文件夹根目录创建docker配置文件命名为**docker-compose.yml**

> 注意此文件名称不可自定义，是docker识别配置文件的唯一手段

按如下配置填写此配置文件：

``` docker
version: '3.8'
services:
  db:
    image: postgres:15
    container_name: star_postgres
    restart: always
    environment:
      POSTGRES_USER: star_user
      POSTGRES_PASSWORD: star_password
      POSTGRES_DB: star_db
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

**步骤4：** 唤出终端，键入下列命令：

``` bash
docker-compose up -d
```

docker会自动拉取，创建并启动此容器

### 特别注意事项

1. 不要手动通过删除文件夹的方式删除镜像，因为**数据卷**依然挂载着，占用磁盘。

2. 不使用时请使用命令：

``` bash
docker-compose stop
```

3. 注意不要使用以下命令：

``` bash
docker-compose down
```

这会导致整个容器都被销毁

4. 请经常使用下面的命令检查容器状态：

``` bash
docker ps
```


## python解释器及虚拟环境配置

请使用**conda**作为环境管理工具：

**步骤1：** 在浏览器搜索conda，并下载适配的版本，Miniconda即可。

> Miniconda更加轻量化，对于个人PC友好，若有足够存储空间也可安装满血版的Conda

**步骤2：** 完成相关配置后进入vscode，打开开发文件夹根目录，并唤出终端，使用conda创建虚拟环境。

```bash
conda create -n venv python=3.13
```

> 其中的venv是此虚拟环境的名称，可以自定义命名其他名称。

> 注意：conda创建环境必须有流畅网络，不要开全局代理

## 数据库可视化——DBeaver

**步骤一：** 下载软件DBeaver，完成后点击左上角新建连接，搜索**PostgreSQL**,按照下面的配置进行输入，请确保与docker的配置文件相同。

- Host: `localhost`
- Port: `5432`
- User: `star_user`
- Password: `star_password`
- Database: `star_db`

**步骤二：** 打开vscode命令行，键入下列命令，安装必要库

``` bash
conda install sqlalchemy psycopg2   
```

**步骤三：** 使用SQLAlchemy连接数据库

新建python文件，输入下面的代码：

``` python
from sqlalchemy import create_engine, text
from sqlalchemy.orm import sessionmaker

# ===================== 请注意这里的 URL =====================
# PostgreSQL 格式（后面我们登录数据库需要这个url来做身份验证）
DATABASE_URL = "postgresql+psycopg2://star_user:star_password@localhost:5432/star_db"
# ============================================================

# 创建数据库引擎
engine = create_engine(DATABASE_URL)

# 创建会话（用来操作数据库）
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

# 测试连接（运行这段代码不报错就是成功！）
def test_connection():
    try:
        db = SessionLocal()
        db.execute(text("SELECT 1"))
        print("✅ Python 连接 Docker 数据库成功！")
        db.close()
    except Exception as e:
        print("❌ 连接失败：", e)

test_connection()
```

在命令行键入下列命令(若尚未激活环境请先激活环境):

``` bash
python .\SQLconnet.py
```
